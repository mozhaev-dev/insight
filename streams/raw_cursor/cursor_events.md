# Table: `cursor_events`

## Overview

**Purpose**: Store individual Cursor IDE usage events including AI model invocations, token usage, and billing information.

**Data Source**: Cursor API via Airbyte connector

---

## Schema Definition

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `unique` | text | PRIMARY KEY | Unique event identifier |
| `kind` | text | NOT NULL | Event type (e.g., "chat", "completion", "agent") |
| `model` | text | NOT NULL | AI model used (e.g., "gpt-4", "claude-3.5-sonnet") |
| `maxMode` | boolean | NOT NULL | Whether max mode was enabled |
| `timestamp` | timestamptz | NOT NULL | Event timestamp |
| `userEmail` | text | NOT NULL | User email address |
| `isHeadless` | boolean | NOT NULL | Whether event was headless (no UI) |
| `isChargeable` | boolean | NOT NULL | Whether event incurs charges |
| `requestsCosts` | numeric | NOT NULL | Request cost in credits |
| `cursorTokenFee` | numeric | NOT NULL | Cursor token fee |
| `isTokenBasedCall` | boolean | NOT NULL | Whether this is a token-based API call |

**Indexes**:
- `idx_cursor_events_timestamp`: `(timestamp)`
- `idx_cursor_events_user_email`: `(userEmail)`

---

## Field Semantics

### Core Identifiers

**`unique`** (text, PRIMARY KEY)
- **Purpose**: Unique identifier for each event
- **Format**: Opaque string ID generated by Cursor
- **Usage**: Primary key, deduplication

**`userEmail`** (text, NOT NULL)
- **Purpose**: Email of the user who triggered the event
- **Format**: Standard email format
- **Example**: "developer@company.com"
- **Usage**: User attribution, activity analysis

### Event Classification

**`kind`** (text, NOT NULL)
- **Purpose**: Type of AI interaction
- **Values**: "chat", "completion", "agent", "cmd-k", etc.
- **Usage**: Event categorization, feature usage analysis

**`model`** (text, NOT NULL)
- **Purpose**: AI model used for the request
- **Examples**: "gpt-4", "gpt-4o", "claude-3.5-sonnet", "cursor-small"
- **Usage**: Model usage statistics, cost analysis

**`maxMode`** (boolean, NOT NULL)
- **Purpose**: Whether the request used Cursor's max mode
- **Values**: true/false
- **Usage**: Feature adoption tracking

**`isHeadless`** (boolean, NOT NULL)
- **Purpose**: Whether the event was triggered without UI interaction
- **Values**: true/false
- **Usage**: Automated vs interactive usage analysis

### Billing

**`isChargeable`** (boolean, NOT NULL)
- **Purpose**: Whether this event counts toward billing
- **Values**: true/false
- **Usage**: Cost tracking, billing analysis

**`requestsCosts`** (numeric, NOT NULL)
- **Purpose**: Cost of the request in credits
- **Format**: Numeric value
- **Usage**: Cost analysis, budget monitoring

**`cursorTokenFee`** (numeric, NOT NULL)
- **Purpose**: Cursor platform token fee
- **Format**: Numeric value
- **Usage**: Fee tracking

**`isTokenBasedCall`** (boolean, NOT NULL)
- **Purpose**: Whether this call is billed by tokens rather than requests
- **Values**: true/false
- **Usage**: Billing model categorization

### Timestamps

**`timestamp`** (timestamptz, NOT NULL)
- **Purpose**: When the event occurred
- **Format**: ISO 8601 timestamp with timezone
- **Usage**: Time-series analysis, activity tracking

---

## Relationships

### Children

**`cursor_events_token_usage`**
- **Join**: `unique` → `event_unique`
- **Cardinality**: One event to one token usage record
- **Description**: Detailed token consumption for each event

## Usage Examples

### Recent events by user

```sql
SELECT
    "unique",
    kind,
    model,
    timestamp,
    requestsCosts
FROM cursor_events
WHERE userEmail = 'developer@company.com'
  AND timestamp > NOW() - INTERVAL '7 days'
ORDER BY timestamp DESC
LIMIT 100;
```

### Model usage statistics

```sql
SELECT
    model,
    kind,
    COUNT(*) as event_count,
    SUM(requestsCosts) as total_cost
FROM cursor_events
WHERE timestamp >= '2026-01-01'
GROUP BY model, kind
ORDER BY event_count DESC;
```

### Daily usage per user

```sql
SELECT
    userEmail,
    DATE(timestamp) as day,
    COUNT(*) as events,
    SUM(CASE WHEN isChargeable THEN 1 ELSE 0 END) as chargeable_events,
    SUM(requestsCosts) as total_cost
FROM cursor_events
WHERE timestamp >= NOW() - INTERVAL '30 days'
GROUP BY userEmail, DATE(timestamp)
ORDER BY day DESC, total_cost DESC;
```

## Notes and Considerations

### Token Usage Details

Token-level details (input/output tokens, cache tokens, costs) are stored in the related `cursor_events_token_usage` table. Join on `unique = event_unique` to get complete event information.

### Event Types

The `kind` field categorizes different types of AI interactions in Cursor IDE. Common values include chat conversations, code completions, agent mode interactions, and cmd-k operations.

### Billing Model

Events can be billed either per-request or per-token (`isTokenBasedCall`). The `isChargeable` flag indicates whether the event actually incurs costs (some may be included in subscription).
